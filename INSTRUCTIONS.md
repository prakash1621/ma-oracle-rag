# M&A Oracle — Setup Instructions

## What This Is

M&A Oracle is a financial due diligence system with:
- **Data ingestion pipeline** — downloads and indexes financial data from 6 sources for 12 tech companies
- **NL2SQL service** — ask financial questions in natural language, get SQL + results
- **RAG pipeline** — routes questions to the best data source (Pinecone, SQLite, Neo4j, or contradiction detection)

## Prerequisites

- Python 3.9 or higher
- pip (Python package manager)
- Internet connection
- Groq API key (free at https://console.groq.com)
- Pinecone API key (free at https://www.pinecone.io)

## Quick Start

### Step 1: Clone and install

```bash
git clone <repo-url>
cd ma-oracle-rag
pip install -r requirements.txt
```

### Step 2: Create your .env file

```bash
cp .env.example .env
```

Open `.env` and fill in:
```
SEC_USER_AGENT=MAOracle yourname@gmail.com
GROQ_API_KEY=your_groq_api_key
PINECONE_API_KEY=your_pinecone_api_key
```

**SEC_USER_AGENT**: SEC requires a contact email for API access. Any email works. Without it, SEC blocks all requests.

### Step 3: Run data ingestion

```bash
python run_ingestion.py
```

Downloads data for all 12 companies from all 6 sources. Takes about 5-10 minutes.

### Step 4: Start the NL2SQL + RAG service

```bash
python -m nl2sql.main
```

Open `http://localhost:8000/` in your browser. You'll see:
- **SQL Mode** — direct financial queries against SQLite (e.g., "Show me Apple's revenue in 2024")
- **RAG Mode** — routes to the best data source automatically (uses mocks until Person 2/3/4 modules are built)

### API Endpoints

| Method | URL | What it does |
|--------|-----|-------------|
| GET | `/` | Web UI (glassmorphism chat interface) |
| GET | `/health` | System status + memory count |
| POST | `/chat` | Direct NL2SQL (SQL mode) |
| POST | `/query` | RAG pipeline (routes to best source) |

## Ingestion Options

### Ingest specific companies

```bash
python run_ingestion.py --tickers AAPL MSFT NVDA
```

### Ingest specific sources

```bash
python run_ingestion.py --sources edgar xbrl transcripts
```

### Available sources

| Source | What it downloads |
|--------|-------------------|
| edgar | SEC 10-K annual report filings |
| xbrl | Structured financial numbers (revenue, income, etc.) |
| transcripts | Quarterly earnings press releases |
| patents | Patent titles and abstracts |
| proxy | Board members, executive compensation |
| company_facts | Company metadata |

### Check what data you have

```bash
python run_ingestion.py --stats
```

## Where Your Data Goes

```
output/
├── edgar/
│   └── chunked_documents.json    ← 10-K filing text chunks
├── xbrl/
│   └── financials.db             ← Financial numbers (SQLite)
├── transcripts/
│   └── chunked_documents.json    ← Earnings release text chunks
├── patents/
│   ├── patents.db                ← Patent data (SQLite)
│   └── chunked_documents.json    ← Patent text chunks
├── proxy/
│   └── chunked_documents.json    ← Proxy statement text chunks
└── company_facts/
```

All text chunks are also indexed in Pinecone (`ma-oracle-cap` index) for vector search.

## Configuration

All settings are in `config.yaml`. Key sections:

- `embedding` — embedding provider (Pinecone integrated)
- `llm` — LLM provider (Groq)
- `vector_store` — Pinecone index settings
- `nl2sql` — NL2SQL service settings (host, port, model, memory)
- `companies` — list of 12 tickers to ingest

Environment variables in `.env` override config.yaml where applicable.

## Troubleshooting

**"SEC_USER_AGENT not set"** — Create `.env` file with your email (Step 2).

**"Incorrect API key"** — Check that `.env` has the right key for the LLM provider. Groq keys start with `gsk_`, OpenAI keys start with `sk-`.

**"PINECONE_API_KEY is required"** — Add your Pinecone key to `.env`.

**Ingestion is slow** — SEC has a rate limit (10 req/sec). Each company takes 30-60 seconds. This is normal.

**NL2SQL not responding** — Check the terminal for errors. Make sure `output/xbrl/financials.db` exists (run ingestion first).

**RAG mode shows mock data** — Expected. Person 2/3/4 modules aren't built yet. The pipeline uses mocks until real modules are merged.
